# 학습 모드

## 개요

SokSok Voca는 4가지 학습 모드(플래시카드, 퀴즈, 예문학습, 오토플레이)와 맞춤학습 기능을 제공한다. 학습 모드는 `study-select/[id]` 화면을 경유하거나 플랜 화면에서 직접 진입할 수 있으며, 세션 종료 후 `study-results` 화면에서 결과를 확인하고 오답을 재학습할 수 있다.

## 관련 파일

- `app/study-select/[id].tsx` — 학습 모드 선택 화면
- `app/flashcards/[id].tsx` — 플래시카드
- `app/quiz/[id].tsx` — 퀴즈
- `app/examples/[id].tsx` — 예문학습
- `app/autoplay/[id].tsx` — 오토플레이
- `app/shadowing/[id].tsx` — 쉐도잉 (준비 중 placeholder)
- `app/study-results.tsx` — 학습 결과 화면
- `components/BatchResultOverlay.tsx` — 배치 완료 오버레이
- `components/StudySettingsModal.tsx` — 학습 설정 모달
- `components/CustomStudyModal.tsx` — 맞춤학습 설정 모달

## 주요 개념

| 개념 | 설명 |
|------|------|
| `StudyResult` | `{ word: Word; gotIt: boolean }` — 세션 내 학습 결과 단위 |
| `studyBatchSize` | 한 배치에 학습할 단어 수 (5/10/20/50/전체). 배치 완료마다 BatchResultOverlay 표시 |
| `filter` param | 학습할 단어 필터: `'all' | 'learning' | 'memorized'` |
| `ids` param | 특정 단어 ID 목록으로 학습 범위 고정 (재학습, 오답 재시도에 사용) |
| `planDay` param | 플랜 연동 시 학습 완료 후 `updatePlanProgress()` 호출에 사용 |

## 동작 시나리오

### 시나리오 1: 학습 모드 선택 화면

1. 단어장 상세 화면 우상단의 "학습 시작" 버튼을 누른다.
2. `study-select/[id]` 화면으로 이동하며 4가지 모드 카드가 표시된다.
   - 플래시카드, 퀴즈, 예문, 쉐도잉
3. 단어가 2개 미만이면 최소 단어 수 경고가 표시된다.
4. 모드 카드를 탭하면 해당 학습 화면으로 이동한다.

---

### 시나리오 2: 플래시카드 학습

1. `flashcards/[id]`로 진입, 단어가 `filter` 조건으로 필터링되어 로드된다.
2. **앞면** 카드에는 단어(term), 발음기호, TTS 버튼, 별표 토글이 표시된다.
3. 카드를 탭하면 3D 뒤집기 애니메이션(Reanimated `rotateY` 0→180도)으로 **뒷면**으로 전환된다.
4. **뒷면**에는 뜻(meaningKr), 예문(exampleEn), 예문 번역(exampleKr), TTS 버튼이 표시된다.
5. 뒷면에서:
   - **오른쪽 스와이프** → "알았다" (gotIt = true)
   - **왼쪽 스와이프** → "다시 볼게" (gotIt = false)
6. 스와이프 임계값(`SCREEN_WIDTH * 0.3`)을 넘으면 `onSwipe(direction)` 실행:
   - "알았다": 해당 단어 `isMemorized = true` 로 표시 (단, 명시적으로만; 설정에 따라 다름)
   - 결과 배열(`results.current`)에 `{ word, gotIt }` 추가
7. `studyBatchSize`만큼 카드를 처리하면 `BatchResultOverlay`가 등장:
   - **다음 배치**: 다음 단어 세트로 진행
   - **이 배치 다시**: 현재 배치 단어를 처음부터 반복
   - **마치기**: 학습 종료 → `study-results`로 이동

> **내부 로직**: `rotation` SharedValue(0→1)로 카드 앞/뒷면의 `backfaceVisibility`와 `opacity`를 제어. `withTiming(duration: 300)`으로 부드러운 전환.

---

### 시나리오 3: 퀴즈 학습

1. `quiz/[id]`로 진입.
2. 퀴즈 타입에 따라:
   - `term-to-meaning`: 단어를 보여주고 뜻 4지선다
   - `meaning-to-term`: 뜻을 보여주고 단어 4지선다
3. 오답 3개는 같은 단어장의 다른 단어에서 무작위 추출 (Fisher-Yates 셔플).
4. 보기를 선택하면:
   - **정답**: 초록 하이라이트 + 성공 햅틱 + 1초 후 다음 문제
   - **오답**: 빨간 하이라이트 + 에러 햅틱 + `incrementWrongCount([wordId])` 호출
   - 정답이면 `resetWrongCount([wordId])` 호출
5. 배치 완료 시 `BatchResultOverlay` 표시.
6. 세션 완료 시:
   - `setWordsMemorized(listId, gotItIds, true)` — 맞춘 단어 암기 처리
   - `saveLastResult(listId)` — 현재 진행도 스냅샷 저장
   - `planDay` param이 있으면 `updatePlanProgress(listId, planDay + 1)` 호출
   - `setStudyResults(results)` 후 `study-results` 화면으로 이동

---

### 시나리오 4: 예문학습

1. `examples/[id]`로 진입.
2. 예문 문장에서 target 단어 부분이 `?` 로 가려진다 (`HighlightedSentence` 컴포넌트).
3. 4개의 보기 중 올바른 단어를 선택한다.
4. `?` 자리를 탭하면 힌트(한국어 뜻)가 표시된다.
5. 정답 선택 후: 단어가 문장에 강조 표시로 드러나고, 예문 번역(exampleKr)이 표시된다.
6. 퀴즈와 동일한 방식으로 배치 처리 및 결과 저장.

> **내부 로직**: `showTerm` 설정이 off이면 문장 내 단어도 숨김. 힌트는 토글 상태(`hintVisible`)로 관리.

---

### 시나리오 5: 오토플레이

1. `autoplay/[id]`로 진입.
2. 단어 카드가 `delay` 설정(1s/2s/3s)에 따라 자동으로 다음 카드로 넘어간다.
3. `TimerRing` 컴포넌트가 남은 시간을 원형 게이지로 시각화한다.
4. 카드 탭 → 뒷면(뜻/예문) 즉시 공개 (`isRevealed = true`).
5. 재생/일시정지 버튼으로 자동 진행을 제어한다.
6. 스와이프로 수동 앞/뒤 이동 가능 (임계값: `SCREEN_WIDTH * 0.2`).
7. `autoPlaySound`가 켜져 있으면 카드가 넘어갈 때마다 TTS 자동 재생.
8. 모든 단어 완료 시 "모두 완료" 화면 표시 → 다시 시작 또는 종료.

> **내부 로직**: `setInterval`으로 타이머 관리. `isRevealed`가 false이면 1.5초 후 자동으로 뒷면 공개. 일시정지 시 `clearInterval` 후 `ringResetKey` 변경으로 TimerRing 리셋.

---

### 시나리오 6: 학습 결과 화면

1. 플래시카드/퀴즈/예문 완료 후 자동으로 `study-results` 화면으로 이동.
2. 결과 화면 표시:
   - 전체 정답이면 트로피 아이콘 + 성공 햅틱
   - 오답이 있으면 체크 아이콘 + 경고 햅틱
   - 정확도(%), 소요 시간, 암기 단어 수, 복습 필요 단어 수
3. 세 가지 버튼:
   - **전체 다시 학습**: 세션의 모든 단어 ID를 `ids` param으로 같은 학습 모드 재시작
   - **오답 다시 학습**: `gotIt = false` 인 단어 ID만으로 재시작 (오답 없으면 버튼 미표시)
   - **학습 종료**: `clearStudyResults()` 후 `router.dismissAll()`

---

### 시나리오 7: 맞춤학습 (Custom Study)

1. 홈 대시보드의 "맞춤학습" 버튼을 누른다.
2. `CustomStudyModal`이 열린다.
3. 설정 가능 항목:
   - 단어장 선택: 전체 또는 특정 단어장(들)
   - 단어장별 Day 선택 (`ListDayPicker`)
   - 단어 필터: 전체 / 학습중 / 오답상위(wrongCount 상위 50개) / 최신(최근 30개) / 별표
   - 학습 모드: 플래시카드 또는 퀴즈
4. "학습 시작" 버튼 → 설정에 맞는 단어 목록 조합 → `flashcards/__custom__` 또는 `quiz/__custom__` 으로 이동.
5. `__custom__` ID를 받은 학습 화면은 `getWordsForList('__custom__')`를 호출하여 모든 표시 단어장의 단어를 합쳐서 반환받는다.

> **내부 로직**: 오답 필터(`wrongCount`)는 `words.sort((a,b) => b.wrongCount - a.wrongCount).slice(0, 50)`. 최신 필터(`recent`)는 `words.sort((a,b) => b.createdAt - a.createdAt).slice(0, 30)`.

---

### 시나리오 8: 학습 설정 모달 (공통)

모든 학습 화면에서 우상단 설정 아이콘으로 접근.

**공통 설정 항목**:
- 단어 필터: 전체 / 학습중 / 암기완료
- 별표 단어만 보기
- 셔플
- 자동 발음 재생
- 표시 항목: 뜻 / 품사 / 발음기호 / 예문 / 예문 번역
- 배치 크기: 5 / 10 / 20 / 50 / 전체
- 퀴즈 타입(퀴즈 모드 전용): 단어→뜻 / 뜻→단어
- 재생 간격(오토플레이 전용): 1초 / 2초 / 3초

## 예외 / 엣지 케이스

- **단어 2개 미만**: 학습 시작 불가 (study-select에서 경고 표시)
- **예문 없는 단어**: 예문학습에서 해당 단어 건너뜀
- **오답 재시도**: 맞춘 단어가 없으면 "오답 다시 학습" 버튼이 표시되지 않음
- **플랜 연동**: `planDay` param으로 진입한 학습에서만 `updatePlanProgress` 호출 (일반 학습에서는 미호출)
- **쉐도잉**: 현재 "준비 중" placeholder 화면만 표시, 기능 미구현
- **배치 재시도**: 배치 재시도는 현재 배치의 단어만 다시 학습 (이전 배치 결과는 유지)
