# 암기계획 시스템

## 개요

암기계획(Plan)은 단어장의 단어들을 날짜별로 분배하여 체계적으로 학습할 수 있게 해주는 기능이다. 전체 단어를 N일에 걸쳐 나눠 학습하는 일정을 생성하고, 매일 오늘의 학습 달성 여부를 추적한다. 홈 대시보드에서 플랜 카드 형태로 진행 상황을 한눈에 확인할 수 있으며, 자정이 지나면 "오늘 달성" 상태가 자동으로 "오늘 학습 필요"로 리셋된다.

## 관련 파일

- `lib/plan-engine.ts` — 플랜 생성, 상태 판정, 일일 학습 상태 계산 핵심 로직
- `app/plan/[id].tsx` — 플랜 설정/관리 화면
- `app/(tabs)/index.tsx` — 홈 대시보드의 플랜 카드 렌더링
- `lib/vocab-storage.ts` — SQLite 플랜 데이터 저장 (`savePlan`, `clearPlan`, `updatePlanProgress`)
- `contexts/VocabContext.tsx` — `setupPlan`, `clearPlan`, `updatePlanProgress` 등 VocabContext 연동

## 주요 개념

| 개념 | 설명 |
|------|------|
| `planTotalDays` | 플랜의 총 학습 일수 |
| `planCurrentDay` | 다음에 학습할 Day 번호 (= 마지막으로 학습한 Day + 1). 기본값 1 |
| `planStartedAt` | 플랜 시작 타임스탬프 (ms). `planEndDate = planStartedAt + planTotalDays * 86400000` |
| `planUpdatedAt` | 마지막으로 학습 완료한 타임스탬프. `NULL`이면 아직 한 번도 학습 안 함 |
| `planFilter` | 플랜 생성 시 선택한 단어 범위: `'all' | 'unmemorized' | 'memorized'` |
| `assignedDay` | 각 단어의 배정 Day 번호. `NULL`이면 플랜 미배정 |
| `PlanStatus` | `'none' | 'in-progress' | 'completed' | 'overdue' | 'inactive'` |
| `StudyState` | `'needs-study' | 'studying' | 'completed'` — 오늘의 학습 달성 상태 |

## 동작 시나리오

### 시나리오 1: 플랜 생성

1. `plan/[id]` 화면으로 진입하면 플랜이 없을 때 설정 모달이 자동으로 표시된다.
2. **하루 목표 단어 수** 입력 (기본값: `suggestWordsPerDay(totalWords)` — 총 단어 수 ÷ 14, 최소 1).
3. **단어 범위** 선택: 전체 / 미암기 / 암기완료.
4. "시작" 버튼을 누르면 `setupPlan(listId, wordsPerDay, filter)` 호출.

> **내부 로직** (`lib/plan-engine.ts:generatePlan`):
> - 선택된 범위의 단어들을 현재 순서 그대로 Day에 순차 배분
> - `day = floor(index / wordsPerDay) + 1`
> - `totalDays = ceil(words.length / wordsPerDay)`
> 
> **SQLite 저장** (`lib/vocab-storage.ts:savePlan`):
> - `lists` 테이블: `planTotalDays`, `planCurrentDay=1`, `planWordsPerDay`, `planStartedAt=now`, `planUpdatedAt=NULL`, `planFilter` 업데이트
> - `words` 테이블: 각 단어에 `assignedDay` 값 설정

5. 플랜 화면에 Day 1의 단어 목록이 표시된다.

---

### 시나리오 2: 플랜 화면에서 학습하기

1. `plan/[id]` 화면에서 현재 Day의 단어 목록이 표시된다.
2. 각 단어 카드를 탭하여 개별 암기 완료 체크 가능 (`toggleMemorized`).
3. "학습 시작" 버튼으로 플래시카드/퀴즈 등 학습 모드 진입.
   - `planDay` 파라미터로 현재 Day를 학습 화면에 전달.
4. 학습 완료 후 `updatePlanProgress(listId, planCurrentDay + 1)` 호출.

> **내부 로직** (`lib/vocab-storage.ts:updatePlanProgress`):
> ```sql
> UPDATE lists SET planCurrentDay = MAX(planCurrentDay, ?), planUpdatedAt = NOW() WHERE id = ?
> ```
> - `MAX()` 로 인해 현재 Day보다 낮은 값으로는 절대 후퇴하지 않음
> - `planUpdatedAt = now` 가 업데이트되는 것이 "오늘 학습했음"의 핵심 신호

---

### 시나리오 3: 홈 대시보드 — 플랜 카드 표시

홈 화면(`app/(tabs)/index.tsx`)에서 모든 플랜 단어장을 `computePlanStatus()`와 `computeDayStudyStatus()`로 계산하여 카드를 표시한다.

**플랜 전체 상태** (`computePlanStatus`, `lib/plan-engine.ts:52`):

| 우선순위 | 조건 | 상태 |
|---------|------|------|
| 1 | `planStartedAt` 없음 | `'none'` |
| 2 | `planUpdatedAt != null AND planCurrentDay > planTotalDays` | `'completed'` |
| 3 | 현재 시각 > 플랜 종료 일시 | `'overdue'` |
| 4 | 마지막 활동으로부터 7일 이상 경과 | `'inactive'` |
| 5 | 나머지 | `'in-progress'` |

**홈 카드 상태 그룹**:
- `'in-progress'` → "진행 중" 섹션
- `'overdue' | 'inactive'` → "지연/중단" 섹션
- `'completed'` → "완료" 섹션 (접을 수 있음)

---

### 시나리오 4: 오늘 학습 상태 계산 (자정 리셋)

`computeDayStudyStatus(list, words, now)` (`lib/plan-engine.ts:133`) 가 매번 `Date.now()`와 `planUpdatedAt`을 비교하여 오늘의 학습 상태를 결정한다.

```
isStudiedToday = isSameCalendarDay(planUpdatedAt, now)
```

**분기 로직**:

1. **`isStudiedToday === true` AND `planCurrentDay > 1`**
   - → `state: 'completed'`, `displayDay: planCurrentDay - 1`
   - 오늘 학습 완료한 경우. "오늘 달성" 초록 배지 표시.

2. **`isStudiedToday === true` AND `planCurrentDay === 1`** (엣지 케이스)
   - → Day 1의 암기 비율로 `'needs-study'` 또는 `'studying'`

3. **`isStudiedToday === false` AND `planCurrentDay > 1`**
   - → `state: 'needs-study'`, `displayDay: planCurrentDay` (다음 학습 Day)
   - 어제 또는 그 이전에 학습했지만 **오늘 아직 학습하지 않은** 경우.

4. **`isStudiedToday === false` AND `planCurrentDay === 1`** (플랜 시작 후 첫날)
   - Day 1부터 cascade 탐색하여 `'needs-study'` / `'studying'` 상태 반환.

> **핵심**: "오늘 달성" 상태는 `planUpdatedAt`이 오늘 날짜인지 여부에만 의존하므로, **자정이 지나면 다음날 앱을 열 때 자동으로 `'needs-study'`로 전환**된다. DB 값은 변경되지 않고 계산 시점의 `now` 값이 달라지는 것.

**StudyState 임계값** (Day 내 암기 비율):
- `dayMemorized === 0` → `'needs-study'`
- `dayMemorized / dayTotal < 0.5` → `'studying'`
- `dayMemorized / dayTotal >= 0.5` → `'completed'`

---

### 시나리오 5: 플랜 재구성 (Rechunk)

학습 중 새로운 단어 수/일로 플랜을 재구성하고 싶을 때.

1. 플랜 화면에서 "다시 구성" 옵션을 선택한다.
2. 새 하루 목표 단어 수를 입력한다.
3. `rechunkPlan(listId, wordsPerDay)` 호출:
   - **미암기 단어만** 필터링 (`!w.isMemorized`)
   - `generatePlan(unmemorized, wordsPerDay)` 로 Day 1부터 재배분
   - 기존 `planCurrentDay`는 1로 초기화됨

---

### 시나리오 6: 플랜 완료

1. 마지막 Day 학습 완료 시 `updatePlanProgress(listId, planTotalDays + 1)` 호출.
2. `planCurrentDay > planTotalDays` 조건 충족.
3. `computePlanStatus()`가 `'completed'` 반환.
4. 홈 화면에서 "완료된 단어장" 섹션으로 이동.
5. 플랜 화면에 트로피 배너 표시.

---

### 시나리오 7: 플랜 삭제

1. 플랜 화면 또는 홈 카드에서 "플랜 삭제" 버튼을 누른다.
2. 확인 후 `clearPlan(listId)` 호출:
   - `lists` 테이블: 모든 plan 관련 컬럼을 초기값으로 리셋
   - `words` 테이블: `assignedDay = NULL` (모든 단어의 플랜 배정 제거)
3. 단어 자체와 암기 상태는 변경되지 않는다.

## 예외 / 엣지 케이스

- **기간 초과(overdue)**: 플랜 종료일이 지났어도 계속 사용 가능. 상태 배너만 "기간 초과"로 변경됨
- **비활성(inactive)**: 7일간 학습 없으면 "중단됨" 상태. 플랜 데이터는 유지되어 다시 학습 가능
- **단어 추가 시 플랜 미반영**: 플랜 생성 후 단어장에 단어를 추가해도 기존 플랜의 `assignedDay` 배정은 자동 업데이트되지 않음 (수동 rechunk 필요)
- **planCurrentDay 역행 방지**: `MAX(planCurrentDay, ?)` SQL로 항상 앞으로만 진행. 이전 Day를 실수로 재학습해도 `planCurrentDay`가 줄어들지 않음
- **당일 달성 후 추가 학습**: `isStudiedToday === true`이면 추가 학습을 해도 "오늘 달성" 상태 유지 (중요: 학습 진행 시간이 오늘이기만 하면 됨)
