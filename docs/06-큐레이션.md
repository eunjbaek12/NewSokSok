# 큐레이션

## 개요

큐레이션은 미리 제작된 단어 모음(공식 프리셋, 커뮤니티 공유, AI 생성)을 탐색하고 자신의 단어장으로 가져오는 기능이다. 공식 큐레이션은 앱 번들에 포함된 정적 데이터이며, 커뮤니티 큐레이션은 다른 사용자들이 공유한 단어장이다. AI 생성 기능은 주제를 입력하면 Gemini API가 관련 단어 10~50개를 생성해준다.

## 관련 파일

- `app/(tabs)/curation.tsx` — 큐레이션 탭 화면 (탐색, AI 생성, 다운로드)
- `constants/curationData.ts` — 앱 번들 내 공식 큐레이션 프리셋 데이터
- `hooks/useThemeGenerator.ts` — AI 주제 단어 생성 훅
- `lib/translation-api.ts` — `generateThemeWords`, `generateMoreWords`
- `lib/ai/gemini-client.ts` — Gemini 클라이언트 직접 호출 모듈
- `server/routes.ts` — `/api/curations` GET/POST/PUT 엔드포인트
- `server/storage.ts` — PostgreSQL 큐레이션 CRUD
- `shared/schema.ts` — `curated_themes`, `curated_words` PostgreSQL 테이블 스키마

## 주요 개념

| 개념 | 설명 |
|------|------|
| 공식 큐레이션 | `constants/curationData.ts`의 `curationPresets` 정적 배열 — 앱 내장 |
| 커뮤니티 큐레이션 | `GET /api/curations`로 가져오는 사용자 공유 단어장 (PostgreSQL 저장) |
| AI 생성 큐레이션 | 주제 입력 → Gemini API 호출 → 단어 목록 생성 (임시, 저장 전 상태) |
| `curated_themes` | PostgreSQL: title, icon, category, level, description, creatorName, creatorId, sourceLanguage, targetLanguage |
| `curated_words` | PostgreSQL: themeId(FK), term, definition, meaningKr, exampleEn |
| `isCurated` | VocaList 필드 — 큐레이션으로 만들어진 단어장 여부 |

## 동작 시나리오

### 시나리오 1: 공식 큐레이션 탐색 및 다운로드

1. 큐레이션 탭 진입 시 기본으로 "공식" 탭이 표시된다.
2. `curationPresets` 배열에서 전체 테마 목록이 로드된다.
3. 언어 필터 칩, 검색창, 보기 방식(상세/간략) 토글을 통해 탐색한다.
4. 테마 카드를 탭하면 상세 뷰로 전환:
   - 테마 아이콘, 제목, 카테고리, 난이도 배지 표시
   - 포함 단어 목록 (term, meaningKr, 예문 미리보기)
   - 단어 개별 선택/전체 선택 체크박스
5. 단어 카드를 탭하면 `WordDetailModal`이 열려 전체 상세 정보 확인 가능.
6. "단어장으로 저장" 버튼을 누른다.
7. 저장 방식 선택:
   - **새 단어장 생성**: `createCuratedList(title, icon, selectedWords)` 호출
   - **기존 단어장에 추가**: `ModalPicker`로 대상 선택 후 `addBatchWords(listId, selectedWords)` 호출
8. 중복 이름이면 자동으로 `-1`, `-2` 접미사 추가 (`getUniqueName()`).

> **내부 로직**: `createCuratedList`는 `createList`와 동일하지만 `isCurated = true`로 저장되어 단어장 카드에 큐레이션 배지가 표시된다.

---

### 시나리오 2: 커뮤니티 큐레이션 탐색

1. 큐레이션 탭에서 "커뮤니티" 탭으로 전환한다.
2. `fetchCloudCurations()` 호출 → `GET /api/curations` 요청.
3. PostgreSQL의 `curated_themes` + `curated_words`를 JOIN하여 응답.
4. 공식 큐레이션과 동일한 탐색/다운로드 UI를 사용한다.

> **내부 로직**: 서버 응답 스키마: `[{ id, title, icon, category, level, description, creatorName, sourceLanguage, targetLanguage, words: [...] }]`

---

### 시나리오 3: AI 주제 단어 생성

1. 큐레이션 탭의 AI 생성 입력창에 주제를 입력한다 (예: "병원에서 사용하는 표현").
2. "생성" 버튼을 누른다.
3. `generateAIWords(query, apiKey, ...)` 호출:
   - 사용자가 설정에서 입력한 `profileSettings.geminiApiKey`로 Gemini 2.0 Flash 직접 호출 (클라이언트 사이드)
   - 프롬프트: `"'${query}' 상황에서 사용할 수 있는 전문적인 영어 단어 10~50개... JSON 배열만 반환"`
   - 반환 형식: `[{term, definition, meaningKr, exampleEn, tags}]`
   - 마크다운 코드 블록(```) 자동 제거 후 파싱
4. 생성된 단어 목록이 선택 UI와 함께 표시된다.
5. 공식 큐레이션 다운로드와 동일한 방식으로 단어장에 저장.

---

### 시나리오 4: 내 단어장 커뮤니티 공유

단어장 공유는 [단어장 관리 문서](./02-단어장-관리.md)의 "단어장 커뮤니티 공유" 시나리오에서 진행되며, 백엔드 처리는 다음과 같다.

1. 클라이언트가 `POST /api/curations` 요청 전송:
   - Body: `{ theme: { title, icon, description, isUserShared, creatorName, creatorId, sourceLanguage, targetLanguage }, words: [{term, definition, meaningKr, exampleEn}] }`
2. 서버에서 `force=true` 아니면 `findDuplicateCuration(creatorId, title)` 중복 확인.
   - 중복이면 `409` 응답 + `existingId` 반환
3. `storage.createCuration(theme, words)`:
   - `curated_themes` INSERT
   - `curated_words` 배치 INSERT (themeId FK)
4. 업데이트 시 `PUT /api/curations/:id`:
   - `creatorId` 일치 확인 (권한 검증)
   - 기존 `curated_words` DELETE 후 새 단어 INSERT

## 예외 / 엣지 케이스

- **사용자가 Gemini API 키를 입력하지 않음**: AI 생성 버튼이 "API Key missing" 에러 알림과 함께 실패
- **Gemini 응답 파싱 실패**: JSON이 마크다운으로 감싸진 경우 자동 추출 시도. 실패 시 에러 표시
- **커뮤니티 탭 오프라인**: `fetchCloudCurations()` 실패 시 빈 배열 반환 (에러 크래시 없음)
- **중복 이름 단어장 저장**: `getUniqueName()`이 자동으로 `-1`, `-2` 접미사를 붙여 저장
- **공식 큐레이션**: 서버와 무관하게 앱 번들에서 로드되므로 오프라인에서도 사용 가능
