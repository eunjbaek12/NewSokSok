# 단어장 관리

## 개요

단어장(VocaList)은 앱의 핵심 데이터 단위다. 사용자는 여러 단어장을 생성하여 단어를 그룹화하고, 순서 변경, 숨기기/보이기, 병합, 커뮤니티 공유 등 다양한 관리 작업을 수행할 수 있다. 단어장 목록 탭과 단어장 상세 화면 두 곳에서 주요 작업이 이루어진다.

## 관련 파일

- `app/(tabs)/vocab-lists.tsx` — 단어장 목록 탭
- `app/list/[id].tsx` — 단어장 상세 화면 (단어 목록, 필터, 편집)
- `contexts/VocabContext.tsx` — 모든 단어장 CRUD 로직
- `lib/vocab-storage.ts` — SQLite CRUD 구현
- `components/ManageModal.tsx` — 단어장 일괄 관리 모달
- `components/ListContextMenu.tsx` — 단어장 카드 컨텍스트 메뉴
- `components/ListCard.tsx` — 단어장 카드 UI 컴포넌트

## 주요 개념

| 개념 | 설명 |
|------|------|
| `VocaList` | 단어장 데이터 모델: id, title, words[], isVisible, position, isCurated, icon, 플랜 관련 필드들 |
| `isVisible` | false이면 단어장 목록에서 숨겨지지만 데이터는 삭제되지 않음 |
| `position` | 정렬 순서. `ORDER BY position DESC` 로 렌더링. 기본값은 생성 시 `Date.now()` |
| `isCurated` | 큐레이션으로 추가된 단어장 여부 |
| `DUPLICATE_LIST` | 같은 이름(대소문자 무관)의 단어장이 있을 때 던지는 에러 코드 |

## 동작 시나리오

### 시나리오 1: 단어장 목록 화면 진입

1. 탭 바에서 "단어장" 탭을 누른다.
2. `VocabContext`의 `lists` 상태에서 `isVisible === true` 인 단어장만 필터링하여 FlatList로 렌더링한다.
3. 각 `ListCard`에는 제목, 단어 수, 암기 진행도(%), 플랜 상태 배지, 마지막 학습 시간(상대시간)이 표시된다.
4. 300px 이상 스크롤하면 하단 우측에 맨 위로 돌아가기 FAB 버튼이 스프링 애니메이션으로 나타난다.

> **내부 로직**: `getListProgress(listId)`가 `{ total, memorized, percent }`를 반환. `computePlanStatus()`로 플랜 상태를 실시간 계산해 배지에 표시.

---

### 시나리오 2: 새 단어장 만들기

1. 목록 화면 우상단의 관리(연필) 버튼을 누르면 `ManageModal`이 열린다.
2. "+ 새 단어장 만들기" 버튼을 누르면 이름 입력 `TextInput`이 나타난다.
3. 이름을 입력하고 확인을 누른다.
4. `createList(title)` 호출:
   - 기존 단어장과 제목이 같은지 대소문자 무관(`.toLowerCase()`)으로 비교한다.
   - 중복이면 `DUPLICATE_LIST` 에러를 throw한다.
   - 중복이 아니면 SQLite `lists` 테이블에 insert하고 `refreshData()` 후 `debouncedSync()`를 실행한다.
5. 새 단어장이 목록 상단에 나타난다.

> **내부 로직**: `position` 값으로 `Date.now()`를 사용하므로 최신 생성 단어장이 `ORDER BY position DESC` 에 따라 최상단에 위치한다.

---

### 시나리오 3: 단어장 이름 변경

1. `ListContextMenu` (카드 롱프레스) 또는 `ManageModal`에서 이름 변경을 선택한다.
2. 다이얼로그에서 새 이름을 입력한다.
3. `renameList(id, newTitle)` 호출:
   - **다른** 단어장과 이름이 겹치는지 확인한다 (`l.id !== id` 조건 포함).
   - 중복 없으면 SQLite `UPDATE lists SET title = ?` 실행.

---

### 시나리오 4: 단어장 순서 변경 (드래그)

1. `ManageModal`에서 리스트 아이템을 길게 눌러 드래그한다.
2. 드롭 후 `reorderLists(orderedIds)` 호출.
3. `orderedIds` 배열의 인덱스에 따라 각 단어장의 `position` 값을 할당 (마지막 인덱스가 가장 낮은 position → 아래에 위치).

---

### 시나리오 5: 단어장 숨기기 / 보이기

1. `ListContextMenu`에서 "숨기기" 또는 `ManageModal`에서 토글 스위치를 조작한다.
2. `toggleVisibility(id)` 호출 → SQLite에서 `isVisible` 값을 0/1로 flip.
3. 숨겨진 단어장은 `vocab-lists.tsx`의 `visibleLists` 필터에서 제외되어 목록에 보이지 않지만, 데이터는 삭제되지 않는다.

---

### 시나리오 6: 단어장 삭제

1. `ListContextMenu`에서 "삭제"를 선택한다.
2. 확인 다이얼로그가 표시된다.
3. 확인하면 `deleteList(id)` 호출 → SQLite `DELETE FROM lists WHERE id = ?` (ON DELETE CASCADE로 연결된 `words`도 자동 삭제).

---

### 시나리오 7: 단어장 병합

1. `ListContextMenu`에서 "병합"을 선택한다.
2. 대상 단어장을 `ModalPicker`로 선택한다.
3. "원본 삭제 여부"를 선택한다.
4. `mergeLists(sourceId, targetId, deleteSource)` 호출.

> **내부 로직** (`lib/vocab-storage.ts`):
> - source 단어장의 단어들을 target 단어장의 기존 단어와 비교 (term 기준 대소문자 무관 비교).
> - 중복되지 않는 단어만 target에 insert.
> - `deleteSource === true`이면 source 단어장을 삭제.

---

### 시나리오 8: 단어장 커뮤니티 공유

1. `ListContextMenu`에서 "공유"를 선택한다.
2. 닉네임 입력 또는 기존 설정 닉네임 확인 후 공유 확인.
3. `shareList(listId)` 호출:
   - `POST /api/curations` 에 theme(제목, 아이콘, 설명, creatorName, creatorId, 언어 정보)와 words(term, definition, meaningKr, exampleEn)를 전송한다.
   - 서버에서 같은 creatorId + 제목 조합이 이미 있으면 `409` 응답 (`DUPLICATE_SHARE` 에러).
4. 409 발생 시 "기존 공유 내용을 업데이트할까요?" 다이얼로그가 표시된다.
   - 업데이트 선택: `PUT /api/curations/:id` 로 재호출 (`force: true`).

---

### 시나리오 9: 단어장 상세 화면 — 단어 필터링 및 정렬

1. `list/[id]`로 진입하면 해당 단어장의 전체 단어가 표시된다.
2. 필터 칩으로 `전체 / 학습중(unmemorized) / 암기완료(memorized)` 전환 가능.
3. 별표 아이콘 토글로 즐겨찾기 단어만 필터링.
4. 정렬은 `최신순 / A-Z / Z-A` 중 선택 가능.

---

### 시나리오 10: 단어장 상세 화면 — 편집 모드 (복수 선택)

1. 우상단 "편집" 버튼을 누르면 편집 모드 진입.
2. 단어 카드를 탭하여 다중 선택 (체크박스 표시).
3. 하단 바에서 세 가지 작업 가능:
   - **삭제**: 선택 단어 일괄 삭제 (`deleteWords`), Snackbar에 "실행 취소" 버튼 표시 (3초간 복원 가능).
   - **복사**: 대상 단어장 선택 후 `copyWords` 호출 (암기/별표 상태 초기화 후 복사).
   - **이동**: 대상 단어장 선택 후 `moveWords` 호출 (기존 단어장에서 제거, 대상에 추가).

> **내부 로직**: 삭제 Snackbar는 `lastAction` 상태에 삭제된 단어 백업을 보관. "실행 취소" 탭 시 `addBatchWords`로 복원.

## 예외 / 엣지 케이스

- **중복 단어장 이름**: `DUPLICATE_LIST` 에러 → UI에 "이미 존재하는 이름입니다" 알림
- **단어가 없는 단어장 공유**: 빈 단어장도 공유 가능 (서버에서 별도 제한 없음)
- **병합 시 전체 중복**: 모든 단어가 중복이면 복사되는 단어 없이 원본만 삭제됨
- **숨겨진 단어장**: 목록에서 보이지 않지만 검색, 클라우드 동기화, 병합 대상으로는 동작함
