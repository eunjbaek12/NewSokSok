# 인증 및 사용자 관리

## 개요

SokSok Voca는 **Google 계정 로그인**과 **게스트 모드** 두 가지 인증 방식을 지원한다. Google 로그인 사용자는 데이터가 클라우드(PostgreSQL)에 동기화되고, 게스트 모드는 기기 로컬(SQLite)에만 저장된다. 인증 상태는 앱 재시작 시에도 AsyncStorage에 유지되어 자동 로그인이 적용된다.

## 관련 파일

- `app/login.tsx` — 로그인 화면 UI
- `contexts/AuthContext.tsx` — 인증 상태 관리 (AuthProvider, useAuth)
- `server/auth.ts` — Google OAuth 처리 및 클라우드 사용자 등록, 동기화 API

## 주요 개념

| 개념 | 설명 |
|------|------|
| `AuthMode` | `'none'` (미로그인) / `'guest'` (게스트) / `'google'` (Google 계정) |
| `GoogleUser` | id, googleId, email, displayName, avatarUrl |
| `@soksok_auth` | AsyncStorage 키 — 인증 상태 영속 저장 |
| `cloud_users` | PostgreSQL 테이블 — Google 사용자 정보 |
| `cloud_vocab_data` | PostgreSQL 테이블 — 사용자 단어장 데이터 JSONB |

## 동작 시나리오

### 시나리오 1: 앱 최초 실행

1. 앱이 시작되면 `AuthProvider`가 AsyncStorage의 `@soksok_auth` 값을 읽는다.
2. 저장된 값이 없으면 `authMode = 'none'`으로 초기화된다.
3. 탭 레이아웃(`app/(tabs)/_layout.tsx`)에서 `authMode === 'none'`을 감지하고 `router.replace('/login')`으로 리다이렉트한다.
4. 로그인 화면이 표시된다.

> **내부 로직**: `AuthProvider`의 `useEffect`에서 `loading = true` 상태로 AsyncStorage를 비동기 읽은 뒤 `loading = false`로 전환. 탭 레이아웃은 `loading`이 `false`가 될 때까지 리다이렉트를 대기한다.

---

### 시나리오 2: Google 로그인 (웹 플랫폼)

1. 사용자가 "Google로 로그인" 버튼을 누른다.
2. `expo-auth-session`의 `AuthRequest`가 Google OAuth 팝업을 열고 `openid profile email` 스코프를 요청한다.
3. 사용자가 Google 계정을 선택하면 OAuth 응답에서 `accessToken`이 반환된다.
4. `loginWithGoogle(accessToken)`이 호출된다.
5. 클라이언트가 `POST /api/auth/google` 에 `{ accessToken }` 을 전송한다.
6. 서버(`server/auth.ts`)가 `https://www.googleapis.com/oauth2/v3/userinfo`에 토큰을 검증한다.
7. Google에서 `sub(googleId)`, `email`, `name`, `picture`를 받아온다.
8. `findOrCreateGoogleUser()`로 PostgreSQL `cloud_users` 테이블에 upsert한다.
   - 신규 사용자이면 `cloud_vocab_data` 테이블에도 빈 배열 `[]` 레코드를 생성한다.
9. 서버가 사용자 객체를 응답하면 클라이언트는 `{ mode: 'google', user }` 를 AsyncStorage에 저장한다.
10. `router.replace('/')` 로 홈 화면으로 이동한다.

> **내부 로직**: OAuth 방식은 `ResponseType.Token` (Implicit Flow). 웹 전용으로 동작하며, 네이티브(iOS/Android)에서는 버튼을 누르면 "준비 중" 알림이 표시된다(`Platform.OS !== 'web'` 분기).

---

### 시나리오 3: 게스트로 시작하기

1. 사용자가 "기기에서 바로 시작" 버튼을 누른다.
2. `loginAsGuest()`가 `{ mode: 'guest', user: null }` 을 AsyncStorage에 저장한다.
3. `router.replace('/')` 로 홈 화면으로 이동한다.
4. 모든 데이터는 기기 로컬 SQLite에만 저장되며, 클라우드 동기화는 수행되지 않는다.

> **내부 로직**: 서버 통신 없이 즉시 완료. `VocabContext`는 `authMode === 'guest'`이면 `syncToCloud()`를 호출하지 않는다.

---

### 시나리오 4: 앱 재시작 시 자동 로그인

1. 앱이 시작되면 `AuthProvider`의 `useEffect`가 AsyncStorage에서 이전 인증 상태를 복원한다.
2. `authMode`가 `'google'` 또는 `'guest'`이면 로그인 화면을 건너뛰고 설정된 시작 탭으로 이동한다.
3. 토큰 갱신이나 서버 재검증은 수행하지 않는다 (저장된 상태를 신뢰).

---

### 시나리오 5: 로그아웃

1. 설정 화면에서 "로그아웃" 버튼을 누른다.
2. 확인 다이얼로그(`ConfirmDialog`)가 표시된다.
3. 확인하면 `logout()`이 `{ mode: 'none', user: null }` 을 AsyncStorage에 저장한다.
4. `router.replace('/login')` 으로 로그인 화면으로 이동한다.
5. 로컬 SQLite 데이터는 삭제되지 않는다 (기기에 그대로 유지).

> **내부 로직**: `logout()`은 단순히 상태를 `'none'`으로 덮어쓴다. 토큰 취소나 서버 세션 종료는 없다.

## 예외 / 엣지 케이스

- **Google 로그인 실패**: 토큰 검증 실패(`401`) 또는 네트워크 오류 시 "로그인에 실패했습니다" 알림 표시 후 로딩 상태 해제
- **`EXPO_PUBLIC_GOOGLE_CLIENT_ID` 미설정**: Google 버튼을 눌러도 "준비 중" 알림만 표시
- **AsyncStorage 읽기 오류**: `try/catch`로 무시하고 `authMode = 'none'` 으로 폴백
- **게스트 → Google 전환**: 현재 마이그레이션 로직 없음. 로그아웃 후 Google 로그인 시 클라우드 데이터가 로컬 데이터를 덮어쓸 수 있음 (Phase 2 병합 미구현)
