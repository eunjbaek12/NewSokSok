# 단어 입력 및 편집

## 개요

단어(Word)는 단어장의 기본 학습 단위다. 단어를 추가하는 방법은 세 가지다: **단일 단어 입력** (자동채움/네이버 사전/AI 지원), **배치 가져오기** (CSV/TSV 텍스트 또는 파일), **사진 가져오기** (카메라/갤러리 + Gemini OCR). 저장된 단어는 상세 화면에서 편집·삭제·복사·이동이 가능하다.

## 관련 파일

- `app/add-word.tsx` — 단어 추가/편집 전체 화면 모달
- `hooks/useAddWord.ts` — 단어 폼 상태 및 자동채움/저장 로직
- `lib/translation-api.ts` — AI 자동채움, 주제 단어 생성
- `lib/naver-dict-api.ts` — 네이버 사전 검색 및 자동완성
- `lib/gemini-api.ts` — 사진 OCR (Gemini Vision)
- `components/BatchImportWorkflow.tsx` — CSV/TSV 배치 가져오기
- `components/PhotoImportWorkflow.tsx` — 사진 가져오기
- `components/WordDetailModal.tsx` — 단어 상세 보기/편집 모달
- `utils/importParser.ts` — CSV/TSV 파싱 유틸리티

## 주요 개념

| 개념 | 설명 |
|------|------|
| `Word` | term(필수), meaningKr(필수), definition, phonetic, pos, exampleEn, exampleKr, tags[], isMemorized, isStarred, wrongCount, assignedDay, sourceLang, targetLang |
| 자동채움(AutoFill) | 단어 입력 후 네이버 사전 → Gemini AI → MyMemory+FreeDictionary 순으로 시도 |
| 자동완성(Autocomplete) | 네이버 자동완성 API → Datamuse API 순으로 시도 |
| `isPendingFill` | React `useTransition`으로 자동채움 중 UI 블로킹 없이 비동기 처리 |

## 동작 시나리오

### 시나리오 1: 단어 한 개 추가 (기본 흐름)

1. 홈 또는 단어장 화면에서 가운데 "+" 탭 버튼을 누른다.
2. `add-word` 전체 화면 모달이 열린다.
3. **term** 필드에 단어를 입력한다 (예: "Serendipity").
4. `enableAutocomplete`가 켜져 있으면 입력 중 네이버/Datamuse 자동완성 드롭다운이 표시된다.
5. "자동 채우기" 버튼(`handleAutoFill`)을 누른다.

> **자동채움 내부 로직** (`useAddWord.ts:runAutoFill`):
> 1. `searchNaverDict(term, sourceLang, targetLang)` 호출
>    - 네이버 사전 API 직접 호출 (`https://{subdomain}.dict.naver.com/api3/{dictCode}/search`)
>    - 결과가 있으면 meaningKr, definition, phonetic, pos, exampleEn, exampleKr를 채우고 **종료**
> 2. 네이버 사전 결과 없으면 `autoFillWord(term, sourceLang, targetLang)` 호출
>    - Gemini 사용 가능(`GET /api/ai/status`)하면 `POST /api/ai/analyze`로 AI 분석
>    - Gemini 불가 시: MyMemory Translation + FreeDictionary API 병렬 호출
> 3. **이미 채워진 필드는 덮어쓰지 않는다** (비어있는 필드만 채움)

6. **meaningKr** (필수) 확인 후 "저장" 버튼을 누른다.
7. `handleSaveWord()`:
   - `term`과 `meaningKr` 둘 다 비어있으면 해당 필드 에러 표시 + 진동 햅틱
   - 유효하면 `addWord(listId, wordData)` → SQLite insert → `refreshData()` → `debouncedSync()`
8. 폼이 초기화되고 다음 단어를 입력할 수 있다.

---

### 시나리오 2: 단어 편집

1. 단어장 상세 화면(`list/[id]`)에서 단어 카드를 탭한다.
2. `WordDetailModal`이 열리고 읽기 모드로 표시된다.
3. "편집" 버튼을 누르면 편집 모드로 전환.
4. 필드 수정 후 저장하면 `updateWord(listId, wordId, updates)` 호출.
   - SQLite `UPDATE words` 실행 + `lastStudiedAt` 갱신

> **편집과 추가의 차이**: `useAddWord` 훅에 `wordId`가 있으면 `updateWord`, 없으면 `addWord`를 호출한다.

---

### 시나리오 3: 배치 가져오기 (CSV/TSV)

1. `add-word` 화면에서 "배치 가져오기" 탭을 선택한다.
2. **입력 단계**: 텍스트를 붙여넣거나 파일(`expo-document-picker`)을 업로드한다.

> **파싱 로직** (`utils/importParser.ts`):
> - 탭 문자(`\t`)가 포함되면 TSV, 아니면 CSV로 자동 감지
> - 열 순서: `term, meaningKr, definition, exampleEn, exampleKr, tags`
> - 첫 행에 'word', 'term', '단어' 등 헤더 키워드가 있으면 자동 건너뜀
> - `term` 또는 `meaningKr`가 없는 행은 `isValid: false` 마킹

3. **미리보기 단계**: 유효한 단어 수와 유효하지 않은 단어 수가 표시된다.
   - 각 행을 개별 편집하거나 유효하지 않은 행을 삭제할 수 있다.
4. "저장" 버튼을 누르면 `addBatchWords(listId, validWords)` 호출.
   - SQLite에 순차적으로 insert (position을 1000씩 감소시켜 최신 항목이 상단에 위치).

---

### 시나리오 4: 사진 가져오기 (Gemini OCR)

1. `add-word` 화면에서 "사진 가져오기" 탭을 선택한다.
2. 카메라 촬영 또는 갤러리에서 이미지를 선택한다 (`expo-image-picker`).
3. 이미지가 base64로 인코딩되어 `fetchWordsFromImage(base64Image)` 호출.

> **Gemini OCR 로직** (`lib/gemini-api.ts`):
> - Gemini 2.5 Flash Lite 모델에 이미지와 프롬프트 전송
> - 프롬프트: "Find English vocabulary words in this image, return as JSON array [{word, meaning, exampleSentence}]"
> - 실패 시 지수 백오프(1s → 2s → 4s)로 최대 3회 재시도
> - HTTP 400(잘못된 이미지)이면 즉시 종료

4. 추출된 단어 목록이 미리보기로 표시된다.
5. 개별 편집 후 저장하면 배치 가져오기와 동일하게 `addBatchWords` 호출.

---

### 시나리오 5: 단어 삭제 / 복사 / 이동 (단어장 상세 화면)

1. 단어장 상세 화면(`list/[id]`)에서 우상단 "편집" 버튼 → 편집 모드 진입.
2. 단어 카드를 탭하여 하나 이상 선택.
3. **삭제**: `deleteWords(listId, wordIds)` → 3초 동안 Snackbar의 "실행 취소"로 복원 가능.
4. **복사**: 대상 단어장 선택(`ModalPicker`) → `copyWords(targetListId, wordIds)`.
   - 대상 단어장에 `isMemorized=false`, `isStarred=false` 로 리셋된 새 단어가 생성된다.
5. **이동**: 대상 단어장 선택 → `moveWords(targetListId, wordIds)`.
   - SQLite에서 `listId` 컬럼만 변경 (암기/별표 상태 유지).

---

### 시나리오 6: TTS로 단어 발음 듣기

1. 단어 추가 화면, 단어 상세 모달, 학습 모드 어디서든 스피커 아이콘을 탭한다.
2. `speak(term, language)` (`lib/tts.ts`) 호출.
   - `expo-speech`가 speed 0.9로 단어를 읽어준다.
   - 이미 재생 중이면 먼저 `stopSpeaking()` 후 새 단어 재생.

## 예외 / 엣지 케이스

- **네이버 사전 지원 언어 쌍**: en-ko, ja-ko, zh-ko, ko-en, ko-ja, ko-zh 6가지만 지원. 미지원 언어 쌍은 AI 폴백으로 넘어감
- **Gemini 미설정**: `GET /api/ai/status`가 `available: false`를 반환하면 AI 자동채움 없이 무료 API만 사용
- **사진 OCR 400 에러**: 이미지 품질이 낮거나 텍스트가 없으면 즉시 실패 처리 (재시도 없음)
- **CSV 파싱 실패**: 빈 파일, 잘못된 형식 → `isValid: false` 행으로 표시, 저장 시 제외됨
- **필수 필드 누락**: term 또는 meaningKr 없으면 저장 불가 + 에러 진동 햅틱
- **음성 인식**: `expo-speech-recognition`은 커스텀 dev 빌드에서만 동작, 일반 빌드에서는 버튼이 비활성화됨
